R-01 Keterlambatan peluncuran sistem dari jadwal yang telah ditetapkan. Perubahan ruang lingkup, keterlambatan vendor, bug kritis. Tinggi Tinggi Tinggi 1. Rencana proyek detail dengan milestone jelas.
2. Rapat koordinasi mingguan.
3. Klausul denda dalam kontrak vendor.
4. Metode agile untuk iterasi cepat. 		Project Manager
R-02 Gangguan jaringan atau server down yang mengakibatkan sistem tidak dapat diakses. Infrastruktur IT tidak memadai, serangan siber, kesalahan konfigurasi. Sedang Sedang Sedang 1. Audit dan upgrade infrastruktur.
2. Sistem redundansi dan backup server.
3. SOP penanganan insiden.
4. Kerja sama dengan provider andal. 		Kepala Bagian IT
R-03 Kurangnya pelatihan staf sehingga pengguna tidak dapat mengoperasikan sistem dengan optimal. Kurang anggaran, jadwal tidak menjangkau semua, materi tidak efektif. Tinggi Tinggi Tinggi 1. Modul pelatihan komprehensif.
2. Pelatihan bertahap dan berulang.
3. Video tutorial dan dokumentasi.
4. Menunjuk "super user" di setiap unit. 		Ketua Tim Implementasi
R-04 Kebocoran data pribadi mahasiswa (NIM, nilai, informasi finansial). Kerentanan keamanan, kesalahan konfigurasi akses, ancaman orang dalam. Tinggi Rendah Tinggi 1. Penetration test dan audit keamanan.
2. Enkripsi data dan prinsip least privilege.
3. Kebijakan keamanan data dan pelatihan.
4. Rencana tanggap darurat kebocoran data. 		CISO / Kepala IT
R-05 Ketidakakuratan data selama proses migrasi dari sistem lama ke sistem baru. Format data tidak kompatibel, script migrasi error, tidak ada validasi. Sedang Sedang Sedang 1. Mapping dan pembersihan data di sistem lama.
2. Trial migrasi dengan sampel data.
3. Validasi dan rekonsiliasi data pasca migrasi.
4. Cadangan data sistem lama. 		Tim Data Management & Vendor
